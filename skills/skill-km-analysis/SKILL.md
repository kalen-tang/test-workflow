# 测试左移接口分析与测试用例推荐

## 技能目标

本技能专门用于测试左移的前置工作，通过分析开发方案文档，提取关键接口信息，并推荐相应的测试用例，帮助测试团队在开发早期阶段介入，提升测试效率和质量。

## 主要功能

### 1. 开发方案文档分析
当用户提供开发方案文档时，我将：
- 全面解析文档结构和内容
- 识别文档中的技术架构信息
- 提取业务需求和功能描述
- 分析系统设计方案
- 检查文档质量
  - 开发方案文档应包含完整的接口设计信息
  - 建议使用标准的API文档格式（如OpenAPI/Swagger）
  - 接口参数说明应详细完整

### 2. 接口信息提取
从开发方案文档中提取以下接口信息，并确保入参出参和接口对应的上：

#### 基础接口信息
- **接口名称**: API 接口的业务名称
- **接口路径**: RESTful API 的 URL 路径
- **请求方法**: HTTP 方法（GET、POST、PUT、DELETE等）
- **功能描述**: 接口实现的业务功能

#### 详细参数信息
- **请求参数**:
  - 路径参数（Path Parameters）
  - 查询参数（Query Parameters）
  - 请求头参数（Headers）
  - 请求体参数（Request Body）
  - 参数类型、是否必填、默认值、取值范围

- **响应参数**:
  - 响应状态码
  - 响应头信息
  - 响应体结构
  - 数据类型和字段说明
  - 错误响应格式

### 3. 测试左移用例推荐

基于提取的接口信息，我将推荐以下类型的测试用例：

#### 功能测试用例
- **正常流程测试**: 验证接口正常业务逻辑
- **边界值测试**: 参数边界值、极限值测试
- **异常流程测试**: 错误参数、异常场景测试
- **业务规则验证**: 业务逻辑正确性验证

#### 接口契约测试用例
- **请求参数校验**: 必填参数、参数格式、参数类型验证
- **响应格式验证**: 响应结构、字段类型、数据格式验证
- **状态码验证**: 各种场景下的HTTP状态码正确性
- **错误处理验证**: 异常情况下的错误响应格式

## 工作流程

### 步骤1：文档接收与预处理
```
接收开发方案文档 → 文档格式识别 → 内容结构化解析 → 文档内容分析 →  检查文档质量
```

### 步骤2：接口信息提取与用例设计
```
接口定义识别 → 参数信息提取 → 信息结构化整理 → 接口风险评估 → 测试场景分析 → 测试用例生成 → 优先级排序
```

### 步骤3：输出与建议
```
只输出一份接口信息提取与用例设计结果文档
```

## 使用指南

1. **上传文档**: 直接提供开发方案文档内容或文档路径
2. **指定分析重点**: 可以指定关注的特定模块或接口
3. **选择输出详细度**: 可以要求详细分析或概要分析
4. **定制化需求**: 可以根据项目特点提出特殊要求

## 最佳实践建议

### 文档质量要求
- 开发方案文档应包含完整的接口设计信息
- 建议使用标准的API文档格式（如OpenAPI/Swagger）
- 接口参数说明应详细完整

### 测试左移策略
- **早期介入**: 在需求评审阶段就开始测试分析
- **持续跟进**: 随开发进度更新测试用例
- **风险驱动**: 重点关注高风险、核心业务接口
- **自动化优先**: 推荐可自动化的测试用例设计

### 协作建议
- 与开发团队保持密切沟通
- 定期更新接口变更信息
- 建立接口测试基线
- 持续优化测试策略

通过本技能的应用，可以显著提升测试左移的效率和质量，帮助团队在开发早期发现和预防潜在问题，降低项目风险。
