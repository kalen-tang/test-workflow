## 🏦 银行标准测试框架说明

本代码模板基于 **pytest_zabank 系列插件**，这是专为银行业务测试定制的标准化框架：

### 核心插件

- **pytest_zabank_wholesale**: 批发银行业务测试插件（提供 lock、BBMScenario、SSOScenario 等）
- **pytest_zati_base**: 基础测试工具插件（提供 data、mysql、xxljob fixtures）
- **pytest_zabank_lns**: 贷款系统测试插件
- **pytest_zabank_momo**: 移动银行测试插件（提供 ElnScenario、LnsScenario 等）

### 框架特性

- ✅ **自动数据加载**: `@pytest.mark.data()` 自动加载 YAML 并注入 `data` fixture
- ✅ **Scenario 模式**: 业务逻辑封装在 Scenario 类，测试方法只传入 Step 对象
- ✅ **自动断言**: Scenario 内部自动处理响应码断言和日志记录
- ✅ **动态渲染**: `data.render()` 支持测试数据动态替换
- ✅ **并发控制**: `@pytest.mark.xdist_group()` 控制测试并发执行

```python
        1. {步骤1}
        2. {步骤2}

        预期结果:
        - {预期1}
        - {预期2}
        """
        # 动态渲染变量（可选）
        data.render({param}=value)

        # 调用 scenario 方法（传入 Step 对象）
        {service}_sc.{method_name}(data[0])
```

## 🔑 关键模式说明

### 模式 1: `@pytest.mark.data()` 装饰器

**核心**: 使用装饰器声明数据文件，框架自动加载 YAML

```python
@pytest.mark.data("account/query_account.yaml")  # ✅ 相对于 data/ 目录的路径
def test_query_account_info(self, data, account_sc):
    """测试方法"""
    pass
```

**错误示例**:

```python
# ❌ 不使用装饰器
@pytest.mark.parametrize("yaml_file", ["account/query_account.yaml"])
def test_xxx(self, yaml_file, account_sc):
    pass
```

### 模式 2: `data` Fixture

**核心**: 直接接收 `data` fixture（框架自动注入）

```python
def test_query_account(self, data, account_sc):
    """
    data: 框架自动加载的 YAML 数据
    - data[0]: 访问第一个 step
    - data[1]: 访问第二个 step
    - data["步骤名称"]: 通过键访问（异常场景）
    """
    step = data[0]  # ✅ 获取第一个步骤
```

**错误示例**:

```python
# ❌ 手动加载 YAML
def test_xxx(self, yaml_file, config, account_sc):
    data = load_yaml(yaml_file, config.env_id)  # ❌ 不需要
```

### 模式 3: `data.render()` 动态渲染

**核心**: 使用 `data.render()` 动态替换变量

```python
def test_query_account(self, data, account_sc, data_pool_sc):
    """动态渲染示例"""
    # 步骤1: 从数据池获取数据
    account_id = self.get_account_id(data, data_pool_sc)

    # 步骤2: 渲染变量到 data 中
    data.render(accountId=account_id)  # ✅ 动态替换 ${accountId}

    # 步骤3: 调用接口（data 中的变量已被替换）
    account_sc.query_account_info(data[0])
```

### 模式 4: 调用 Scenario 方法

**核心**: 传入 `Step` 对象（不是 Model 对象）

```python
# ✅ 正确：传入 Step 对象
account_sc.query_account_info(data[0])

# ❌ 错误：传入 Model 对象
params = model.QueryAccountInfo()
params.json_data = {...}
account_sc.query_account_info(params)  # ❌ 不需要
```

```python
@pytest.mark.data("resource/create_resource.yaml")

    预期结果:
    - 两个接口都返回成功
```

## 🎯 Allure 注解使用

### 完整注解示例

```python
@allure.epic("账户管理接口")           # ✅ 模块级别
@pytest.mark.P1                      # ✅ 优先级
@pytest.mark.xdist_group("account_group")  # ✅ 并发控制
class TestAccount:

    @pytest.mark.data("account/query_account.yaml")
    @allure.title("查询账户信息")  # ✅ 测试标题
    def test_query_account_info(self, data, account_sc):
        """测试方法"""
        pass
```

### 注解说明

| 注解 | 作用 | 示例 |
|-----|-----|-----|
| `@allure.epic()` | 项目模块（类级别） | `"账户管理接口"` |
| `@allure.title()` | 测试标题（方法级别） | `"查询账户信息"` |
| `@pytest.mark.P0/P1/P2` | 优先级标记 | P0（核心），P1（重要），P2（辅助） |
| `@pytest.mark.xdist_group()` | 并发分组 | `"account_group"`（同组串行执行） |
| `@pytest.mark.data()` | 数据文件路径 | `"account/query_account.yaml"` |
