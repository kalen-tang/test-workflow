
# 需求文档分析工具

智能需求文档分析工具 - 支持中文繁简转换，输出标准SRS格式的规范化需求文档，便于后续测试用例生成。

## 核心特性

### 🔤 智能语言处理能力
- **繁简识别转换**：自动识别中文简体/繁体输入，确保输出为简体中文
- **专业术语保护**：保持技术术语和专业名词的一致性，避免翻译失真
- **领域语言理解**：具备金融、软件工程等专业领域的语义理解能力

### 📄 文档输入支持格式
- **支持的格式**：直接粘贴文本、上传Word(.doc/.docx)、PDF、ODT文档
- **在线文档处理**：支持Confluence、在线文档链接的直接解析
- **结构化识别**：自动识别表格、列表、章节等文档结构元素
- **多格式兼容**：统一处理不同来源和格式的文档输入

### 📋 深度需求分析能力
- **功能识别策略**：基于语义分析识别功能点，提取业务逻辑和技术要求
- **细节深度挖掘**：从描述性文本中提取具体的业务规则、约束条件、数据要求
- **业务流程还原**：识别并重构用户流程、系统交互、数据处理流程
- **质量维度覆盖**：性能、安全、兼容性、可靠性等多维度需求识别

### 🎯 标准化输出规范
- **SRS标准遵循**：严格按照软件需求规格说明书国际标准格式
- **内容结构化**：确保需求描述、规则说明、验收标准的结构清晰
- **测试可转化**：验收标准采用场景描述格式，便于直接转换为测试用例
- **技术可实现**：包含具体的技术要点、数据模型、性能指标要求

## 使用方式

```
请分析这个Word文档并生成标准需求规格说明:   
    [上传Word文档]
    或[读取/script 文件夹下的指定文件]
```

## 工作流程

### 1、文档获取
```
获取并解析需求文档：
   [上传Word文档]
   或[读取/script 文件夹下的指定文件]
```

### 2、文档内容提取
```
请分析需求文档
    - 全面解析文档结构和内容，提取业务需求、功能描述
    - 如果文档中存在图片，请一并分析，如无法解析，请保留原始内容 
    - 如果需求内容存在表格，请分析并提取表格中的【场景】、【说明】两部分，作为需求内容继续进行智能分析 
    - 如果文档存在附件或者超链接，保留原始内容，作为输出的附件章节
```

### 3、需求分析
```
根据第2个步骤提取完整的需求内容，进行需求分析，提取需求清单、需求描述和细节
```

### 4、质量检查
```
请检查这份需求文档的完整性和可测试性，并对这份需求文档的质量并提出改进建议
[粘贴现有需求文档]
```
### 5、输出
```
按照标准需求规格说明，只输出一份标准SRS格式且为中文简体的需求文档
存放目录：\wf_bank_test\script\
```

## 技术特点

### 🧠 智能分析引擎
- **深度语义解析**：理解上下文关系，识别功能点的业务逻辑和技术约束
- **结构化信息提取**：自动识别需求文档中的表格、列表，提取场景描述和技术要求
- **领域知识应用**：结合金融、软件工程等专业知识，理解专业术语和流程规范
- **多维度分类**：基于内容特征自动分类需求类型（功能性、非功能性、业务需求）

### 🔧 质量保证机制
- **完整性验证**：检查需求文档是否包含背景、目标、功能、验收标准等核心要素
- **一致性校验**：确保术语使用一致，避免同一概念在不同章节使用不同表述
- **可测试性评估**：分析验收标准是否具备可测试性，提出改进建议
- **逻辑连贯性**：验证业务逻辑的连贯性，识别流程断点和矛盾之处

### 📊 分析深度控制
- **层级化解析**：从整体架构到具体细节的渐进式分析策略
- **业务规则挖掘**：从描述性文本中提取具体的约束条件、边界条件、例外处理
- **技术实现识别**：识别隐藏的技术要求，如性能指标、安全机制、数据模型
- **关联性分析**：建立需求之间的依赖关系和优先级关联

**需求分析深度要求：**
- 深入分析原始文档的每一章节，提取具体的技术实现要求
- 识别隐含的业务规则和约束条件，确保逻辑流程的连贯性
- 保留原始文档的专业术语和细节，确保技术准确性
- 验证需求的完整性和可实施性，提供改进建议
