## 三、接口信息汇总

### 接口1：用户登录

- **所属微服务**：zabank_user_service
- **接口路径**：zabank_user_service/user/login
- **请求方法**：POST
- **功能描述**：用户通过手机号和密码登录系统
- **关联需求**：F001（需求文档）

#### 请求参数

**JSON 格式**：
```json
{
  "phone": "string",
  "password": "string"
}
```

**参数说明**：

| 参数名 | 类型 | 必填 | 说明 | 示例值 | 业务规则 |
|--------|------|------|------|--------|---------|
| phone | string | 是 | 手机号 | "13800138000" | BR001（11位数字，以1开头） |
| password | string | 是 | 密码 | "Test1234" | BR002（8-20位，包含字母和数字） |

#### 响应参数

**JSON 格式**：
```json
{
  "code": 200,
  "message": "success",
  "data": {
    "userId": "string",
    "token": "string",
    "nickname": "string"
  }
}
```

**参数说明**：

| 字段名 | 类型 | 说明 | 示例值 |
|-------|------|------|-------|
| code | int | 状态码 | 200 |
| message | string | 响应消息 | "success" |
| data.userId | string | 用户ID | "U123456" |
| data.token | string | 登录令牌 | "eyJhbGciOiJIUzI1..." |
| data.nickname | string | 用户昵称 | "张三" |

#### 场景2：忘记密码重置流程（US002 + AC004 + AC005）

**场景来源**：
- 需求文档-US002（用户故事）
- 需求文档-AC004（验收标准-获取验证码）
- 需求文档-AC005（验收标准-重置密码）

**场景描述**：
作为忘记密码的用户，我希望通过短信验证码重置密码，以便快速恢复账号访问。

**涉及接口**：
1. POST /user/getVerifyCode - 获取验证码
2. POST /user/resetPassword - 重置密码
3. POST /user/login - 登录验证（验证新密码）

**接口调用关系图**：
```
getVerifyCode（获取验证码） → resetPassword（重置密码） → login（验证新密码）
      ↓                            ↓                           ↓
  返回codeId                  使用codeId和验证码           使用新密码登录
```
