# 测试用例设计模式

## 单接口测试用例

### 1. 正常流程测试

验证接口正常业务逻辑。

**设计要点**：
- 使用有效的参数组合
- 验证预期的成功响应
- 检查返回数据的完整性和正确性

**示例**：
```markdown
##### 功能测试
1. **正常场景测试**
   - 用例描述：使用有效参数创建活动
   - 前置条件：用户已登录，具有创建权限
   - 测试步骤：
     - 步骤1：调用创建接口，传入有效参数
   - 请求参数：
   ```json
   {
     "activityName": "测试活动",
     "startTime": "2026-03-10 00:00:00",
     "endTime": "2026-03-20 00:00:00"
   }
   ```
   - 预期结果：
   ```json
   {
     "code": 200,
     "data": {
       "activityId": "ACT123456",
       "status": "CREATED"
     }
   }
   ```
```

### 2. 边界值测试

参数边界值、极限值测试。

**设计要点**：
- 测试最小值、最大值
- 测试边界条件（如日期范围、数量限制）
- 验证边界值的处理是否正确

**示例**：
```markdown
##### 边界值测试
1. **参数最小值测试**
   - 用例描述：测试参数为最小值时的处理
   - 请求参数：
   ```json
   {
     "quantity": 1  // 最小购买数量
   }
   ```
   - 预期结果：接口正常返回

2. **参数最大值测试**
   - 用例描述：测试参数为最大值时的处理
   - 请求参数：
   ```json
   {
     "quantity": 9999  // 最大购买数量
   }
   ```
   - 预期结果：接口正常返回或返回超出限制提示
```

### 3. 异常流程测试

错误参数、异常场景测试。

**设计要点**：
- 必填参数缺失
- 参数类型错误
- 参数格式错误
- 参数取值不合法
- 验证错误响应格式

**示例**：
```markdown
##### 异常测试
1. **必填参数缺失**
   - 用例描述：缺少必填参数
   - 请求参数：
   ```json
   {
     "activityName": null  // 必填参数为空
   }
   ```
   - 预期结果：
   ```json
   {
     "code": 400,
     "message": "参数错误：activityName不能为空"
   }
   ```

2. **参数类型错误**
   - 用例描述：传入错误的参数类型
   - 请求参数：
   ```json
   {
     "quantity": "abc"  // 应为数字，传入字符串
   }
   ```
   - 预期结果：
   ```json
   {
     "code": 400,
     "message": "参数类型错误：quantity应为数字"
   }
   ```
```

### 4. 业务规则验证

业务逻辑正确性验证。

**设计要点**：
- 验证业务规则是否正确执行
- 检查业务状态的正确性
- 验证业务约束条件

**示例**：
```markdown
##### 业务规则验证
1. **活动时间校验**
   - 用例描述：结束时间早于开始时间
   - 请求参数：
   ```json
   {
     "startTime": "2026-03-20 00:00:00",
     "endTime": "2026-03-10 00:00:00"  // 早于开始时间
   }
   ```
   - 预期结果：
   ```json
   {
     "code": 400,
     "message": "结束时间不能早于开始时间"
   }
   ```
```

## 接口契约测试用例

### 1. 请求参数校验

必填参数、参数格式、参数类型验证。

**测试点**：
- 所有必填参数是否生效
- 参数格式是否正确验证（如日期格式、邮箱格式）
- 参数类型是否正确验证（如数字、字符串、布尔）

### 2. 响应格式验证

响应结构、字段类型、数据格式验证。

**测试点**：
- 响应结构是否符合文档定义
- 字段类型是否正确
- 数据格式是否规范（如日期格式、金额格式）

### 3. 状态码验证

各种场景下的 HTTP 状态码正确性。

**常见状态码**：
- `200`：成功
- `400`：请求参数错误
- `401`：未认证
- `403`：无权限
- `404`：资源不存在
- `500`：服务器内部错误

### 4. 错误处理验证

异常情况下的错误响应格式。

**验证点**：
- 错误响应包含错误码
- 错误响应包含错误信息
- 错误信息清晰易懂
