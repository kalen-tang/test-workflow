```python
def scan_existing_test_data(data_dir: str = "data") -> dict:
    """
    扫描现有YAML文件,提取测试账号信息

    Returns:
        dict: {
            "sit": [{"user_id": "xxx", "customer_no": "yyy", "source": "file_path"}, ...],
            "auto_qe": [...],
            "uat": [...]
        }
    """
    import yaml
    import glob

    accounts = {"sit": [], "auto_qe": [], "uat": []}

    # 扫描所有YAML文件
    yaml_files = glob.glob(f"{data_dir}/**/*.yaml", recursive=True)

    for file_path in yaml_files:
        with open(file_path, 'r', encoding='utf-8') as f:
            try:
                data = yaml.safe_load(f)

                # 提取env配置中的账号
                if "env" in data:
                    for env in ["sit", "auto_qe", "uat"]:
                        if env in data["env"]:
                            env_data = data["env"][env]

                            # 提取user_id和customer_no (可能有多种变量名)
                            user_id = None
                            customer_no = None

                            for key, value in env_data.items():
                                if "user_id" in key.lower():
                                    user_id = value
                                if "customer_no" in key.lower():
                                    customer_no = value

                            if user_id and customer_no:
                                accounts[env].append({
                                    "user_id": user_id,
                                    "customer_no": customer_no,
                                    "source": file_path
                                })
            except Exception as e:
                # 解析失败,跳过
                continue

    return accounts
```

## 总结

### Service层缺失处理策略

| 场景 | 推荐方案 | 优点 | 缺点 |
|-----|---------|------|------|
| 接口定义清晰 | 自动创建 (方案A) | 效率高,无需手动操作 | 可能需要微调生成的代码 |
| 接口定义模糊 | 提示手动添加 (方案B) | 确保代码质量 | 需要手动操作,效率较低 |

### 测试数据缺失处理策略

| 场景 | 推荐方案 | 优点 | 缺点 |
|-----|---------|------|------|
| 有现成数据可复用 | 从现有YAML复制 (方案B) | 最快捷 | 可能需要验证数据有效性 |
| 无现成数据 | 使用占位符 (方案A) | 明确提示,不易遗漏 | 需要后续手动填充 |
| 需要特定状态账号 | 生成数据准备指南 | 指导详细,易于执行 | 需要额外时间准备数据 |

### 关键原则

1. **自动化优先**: 能自动创建的尽量自动创建
2. **明确提示**: 无法自动化时,提供清晰的手动操作指南
3. **数据复用**: 优先复用现有测试数据,避免重复劳动
4. **质量保证**: 自动生成的代码需提示用户检查和确认
