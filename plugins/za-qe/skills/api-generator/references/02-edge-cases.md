# 边界情况处理指南

**版本**: 2.0（基于银行标准测试框架）
**更新日期**: 2026-03-19
**适用框架**: pytest_zabank 系列插件（pytest_zabank_wholesale、pytest_zati_base 等）

---

## 🏦 核心原则

本指南基于 **pytest_zabank 系列插件**（银行标准测试框架）,处理以下边界情况:

1. **Service 层缺失**: Controller 层没有对应的接口方法
2. **测试数据缺失**: 项目中没有可用的测试账号数据

**关键假设**:
- ✅ 测试代码使用 `@pytest.mark.data()` 装饰器
- ✅ YAML 格式使用 `variables` 而非 `env` 多环境配置
- ✅ Scenario 方法接收 Step 对象（来自 `data[index]`）
- ✅ Fixture 使用 `function` scope 并依赖 `env` 和 `lock`

## 情况1: Service层缺失接口方法

### 问题描述

生成测试用例时,发现 `service/{service_name}/__init__.py` 中没有对应的接口映射。

**表现**:
- 扫描 `service/{service_name}/__init__.py` 的 mapping 字典
- 未找到匹配的接口方法名
- 无法确定对应的 Model 类

### 解决方案A: 自动创建 Service 层代码（推荐）

#### 步骤1: 读取文档中的接口定义

```markdown

- **接口路径**: /account/query
- **请求方法**: POST
- **功能描述**: 查询账户基本信息

### 请求参数
| 参数名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| accountId | String | 是 | 账户ID |
```

## 常见问题

### Q1: 测试账号已失效怎么办?

A: 重新执行数据获取流程,更新 YAML 文件中的账号数据。

### Q2: 可以所有测试用例共用一个账号吗?

### Q3: 占位符可以不替换直接运行吗?

A: 不可以。占位符不是有效的测试数据,会导致测试失败。必须替换为真实账号后才能运行。

### 解决方案B: 尝试从现有数据中推断

#### 步骤1: 扫描现有 YAML 文件

```python
def scan_existing_test_data(data_dir: str = "data") -> dict:
    """
    扫描现有YAML文件,提取测试账号信息

    返回:
    - dict: {
        "accounts": [
            {
                "username": "xxx",
                "account_id": "yyy",
                "source": "file_path"
            },
            ...
        ]
    }
    """
    import yaml
    import glob

    accounts = []

    # 扫描所有YAML文件
    yaml_files = glob.glob(f"{data_dir}/**/*.yaml", recursive=True)

    for file_path in yaml_files:
        with open(file_path, 'r', encoding='utf-8') as f:
            try:
                data = yaml.safe_load(f)

                # 提取variables配置中的账号
                if "variables" in data:
                    variables = data["variables"]

                    # 提取账号信息（可能有多种变量名）
                    account_info = {}

                    for key, value in variables.items():
                        if "user" in key.lower() or "account" in key.lower():
                            account_info[key] = value

                    if account_info:
                        account_info["source"] = file_path
                        accounts.append(account_info)

            except Exception as e:
                # 解析失败,跳过
                continue

    return {"accounts": accounts}
```

## 总结

### Service层缺失处理策略

| 场景 | 推荐方案 | 优点 | 缺点 |
|-----|---------|------|------|
| 接口定义清晰 | 自动创建（方案A） | 效率高,无需手动操作 | 可能需要微调生成的代码 |
| 接口定义模糊 | 提示手动添加（方案B） | 确保代码质量 | 需要手动操作,效率较低 |

### 测试数据缺失处理策略

| 场景 | 推荐方案 | 优点 | 缺点 |
|-----|---------|------|------|
| 有现成数据可复用 | 从现有YAML复制（方案B） | 最快捷 | 可能需要验证数据有效性 |
| 无现成数据 | 使用占位符（方案A） | 明确提示,不易遗漏 | 需要后续手动填充 |
| 需要特定状态账号 | 生成数据准备指南 | 指导详细,易于执行 | 需要额外时间准备数据 |

### 关键原则

1. **自动化优先**: 能自动创建的尽量自动创建
2. **明确提示**: 无法自动化时,提供清晰的手动操作指南
3. **数据复用**: 优先复用现有测试数据,避免重复劳动
4. **质量保证**: 自动生成的代码需提示用户检查和确认
