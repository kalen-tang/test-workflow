# 第 2 章 专业术语表（避免歧义）

|  |  |
| --- | --- |
| 名称 | 说明 |
| DRP | 债务舒缓计划 Debt Relief Plan，由债务人及个别债权人直接协商贷款重组的还款方案。 |
| IVA | 个人自愿安排Individual Voluntary Arrangement。债务人将委任律所为代理人，代理人拟定债务重组方案，在经过法庭聆讯及颁布命令后，代理人确保债务人的还款按重组计划分派给债权人。  DRP和为IVA的最大分别在于，DRP无需透过法律程序，可以与个别债权人直接协商新的还款方案。 |
| IDRP | 综合债务舒缓计划Interbank Debt Relief Plan。当债务人有多个债权人时，债务人只需与最大债权人商谈重组方案，其他债权人的欠款都合并到最大债权人之下。债务人还款后，由最大债权人向其他债权人分配还款。  IDRP与DRP的最大分别在于，IDRP可以一揽子将在不同债权人的欠款合并重组。 |
| Lead Participant（LP） | 在IDRP中，一般由最大债权人承担Lead Participant的角色。Lead Participant负责代理其他银行的债权，与客户统一协商重组方案。在还款阶段，Lead Participant负责接收客户的全部还款，再分配至其他银行。 |
| IDLP  IDP | 在IDRP中，由于ZA是否为Lead Participant对系统处理有较大影响，为以示区分，IDLP指ZA为Lead Participant的重组，IDP指ZA非Lead Participant的重组。  两者均为与IVA/DRP并列的重组类型。 |
| 母借据 | Master Loan Account，ZA是Lead Participant时，记录重组后客户全部欠款信息的借据。 |
| 子借据 | Subsidiary Loan Account，ZA是Lead Participant时，记录重组后客户在各个债权人欠款信息的借据。 |

## App版本更新机制

由于外部存在多个App原生版本，在本需求上线时，为了确保所有Scameter高风险转账全部被拒绝，需要进行以下更新：

|  |  |  |
| --- | --- | --- |
| **版本号** | **更新策略** | **客户体验及备注** |
| V3.7.0及更早的App版本 | 强制升级原生版本 | 打开App后即弹窗提示需要升级到最新原生版本，更新前无法使用任何功能 |
| V3.7.1（5月初版本）~ V3.7.4  需要测试 4个版本\*2类设备（Android+iOS}） | 强制热更新 | App进入转账功能后即强制进行热更新，  -Loading成功后进入最新版本功能，即拒绝Sacmeter高风险转账  -如果发生极小概率的Loading更新失败，用户转账给高风险用户后需要填写问卷，无论填写内容如何，转账都无法进行 |
| V3.7.5（06.17迭代将发布的最新版本号） | 原生发布，预计06.23左右在AppStore等应用市场过审后全面放开 | App拒绝所有Sacmeter高风险转账 |
