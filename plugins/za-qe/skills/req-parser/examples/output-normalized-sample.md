# 第 3 章 业务逻辑与功能需求

## 3.1 关键业务流程

用户注册 → 手机号验证 → 设置密码 → 注册成功 → 登录

用户登录 → 输入手机号和密码 → 验证 → 登录成功/失败

密码重置 → 输入手机号 → 获取验证码 → 输入新密码 → 重置成功

## 3.2 功能需求

### 1. 用户登录

适用范围：全端

用户可以通过手机号和密码登录系统。这是系统的核心功能之一，必须确保安全性和易用性。

业务规则：
- 手机号必须为11位数字，且以1开头
- 密码长度必须为8-20位，包含字母和数字
- 密码不能包含手机号
- 用户连续输错密码3次，账号锁定30分钟
- 同一手机号只能注册一个账号

### 2. 密码重置

适用范围：全端

用户忘记密码时，可以通过短信验证码重置密码。

业务规则：
- 验证码为6位数字
- 验证码5分钟内有效，超时需要重新获取
- 同一手机号每天最多发送10次验证码

# 第 5 章 验收标准

| **功能点** | **验收标准** |
|---|---|
| 用户登录 | 1. 给定一个已注册用户，当输入正确的手机号和密码并点击登录时，那么应跳转到首页并显示用户昵称 2. 给定一个已注册用户，当输入错误密码并点击登录时，那么应停留在登录页并显示错误提示"密码错误" 3. 给定一个已连续输错密码2次的用户，当第3次输入错误密码时，那么应显示"账号已锁定30分钟" |
| 密码重置 | 1. 给定一个已注册用户，当输入手机号并点击获取验证码时，那么应收到6位数字验证码 2. 给定一个已获取验证码的用户，当输入正确验证码和符合规则的新密码时，那么应显示"密码重置成功"并跳转到登录页 3. 给定一个验证码已超过5分钟的用户，当输入该验证码时，那么应显示"验证码已过期，请重新获取" |
