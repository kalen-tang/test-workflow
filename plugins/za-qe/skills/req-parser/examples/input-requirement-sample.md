# 用户管理模块需求文档 V1.2

**文档版本**: V1.2
**创建日期**: 2026-03-10
**负责人**: 产品部

---

## 1. 功能概述

本模块实现用户注册、登录、权限管理等核心功能，旨在提升用户体验，降低运营成本。

---

## 2. 用户登录

### 2.1 功能描述

用户可以通过手机号和密码登录系统。这是系统的核心功能之一，必须确保安全性和易用性。

### 2.2 验收标准

- 用户输入正确的手机号和密码，点击登录按钮，跳转到首页
- 用户输入错误的密码，显示错误提示"密码错误"
- 用户连续输错密码3次，账号锁定30分钟

### 2.3 接口设计

登录接口：`POST /user/login`

参数：
- mobile：手机号（必填）
- password：密码（必填）
- remember_me：记住密码（可选）

---

## 3. 密码重置

### 3.1 功能描述

用户忘记密码时，可以通过短信验证码重置密码。

### 3.2 验收标准

- 用户输入手机号，点击"获取验证码"，收到6位数字验证码
- 用户输入正确的验证码和新密码，密码重置成功
- 验证码5分钟内有效，超时需要重新获取

### 3.3 接口设计

重置密码接口：`POST /user/reset-password`

---

## 4. 业务规则

### 4.1 手机号规则

- 手机号必须为11位数字，且以1开头
- 同一手机号只能注册一个账号

### 4.2 密码规则

- 密码长度必须为8-20位
- 密码必须包含字母和数字
- 密码不能包含手机号

### 4.3 安全规则

- 用户连续输错密码3次，账号锁定30分钟
- 验证码5分钟内有效
- 验证码同一手机号每天最多发送10次

---

## 5. 字段定义

| 字段名 | 类型 | 是否必填 | 格式要求 | 说明 |
|--------|------|----------|----------|------|
| mobile | string | 是 | 11位数字 | 手机号 |
| password | string | 是 | 8-20位，字母+数字 | 密码 |
| remember_me | boolean | 否 | true/false | 记住密码 |
| sms_code | string | 是（重置密码时） | 6位数字 | 短信验证码 |

---

## 6. 非功能需求

### 6.1 性能要求

- 登录接口响应时间 < 500ms
- 验证码发送时间 < 3秒

### 6.2 安全要求

- 密码需加密传输（HTTPS）
- 密码需加密存储（BCrypt）
- 登录失败记录需记录日志
