# 代码变更质量风险识别模式

## 风险等级定义

| 等级 | 说明 | 对测试的影响 |
|------|------|-------------|
| P0 - 高风险 | 可能导致数据错误、服务不可用、安全漏洞 | 必须覆盖，阻塞发布 |
| P1 - 中风险 | 可能导致功能异常、性能下降 | 建议覆盖，影响用户体验 |
| P2 - 低风险 | 影响范围小，不影响核心功能 | 可选覆盖，回归测试包含 |

---

## P0 高风险模式

### 1. 数据库 Schema 变更

**识别特征**：
- 文件路径：`*.sql`、`*migration*.java/.go`、`*V*.*.sql`（Flyway）、`db/migrate/*.rb`
- 变更内容包含：`ALTER TABLE`、`ADD COLUMN`、`DROP COLUMN`、`CREATE INDEX`、`DROP INDEX`、`MODIFY COLUMN`

**风险点**：
- 新增非空字段未提供默认值 → 存量数据写入失败
- 删除字段但代码仍有引用 → 运行时错误
- 索引变更影响查询性能 → 慢查询、超时
- 字段类型变更（如 VARCHAR→INT）→ 数据截断或转换错误

**测试建议**：
- 在存量数据（SIT 数据库）验证迁移脚本执行
- 验证迁移后应用正常读写
- 检查回滚脚本的可用性

### 2. 接口契约变更

**识别特征**：
- Controller/Router 文件变更（`*Controller.java`、`handler.go`、`router.go`）
- 新增/删除/重命名请求参数、响应字段
- HTTP Method 或 URL Path 变更

**风险点**：
- 删除字段 → 下游调用方 NPE 或解析失败
- 新增必填参数 → 已有调用方报参数缺失错误
- 修改字段类型（String→Long）→ 下游强类型语言解析失败

**测试建议**：
- 对比接口变更前后的 Swagger/API 文档
- 测试所有已知调用方的兼容性
- 验证新字段的边界值和必填校验

### 3. 核心业务逻辑修改

**识别特征**：
- 文件路径：`*Service*.java`、`*Domain*.java`、`*Biz*.go`
- 涉及：金额计算、余额变更、积分/权益逻辑、风控规则

**风险点**：
- 计算公式变更 → 金额误差
- 条件判断调整 → 部分用户场景被遗漏
- 状态机转换变更 → 订单/流程卡单

**测试建议**：
- 准备边界值测试数据（0、负数、最大值、小数精度）
- 覆盖所有业务规则分支
- 与需求 AC（验收标准）逐条对比验证

### 4. 并发与事务变更

**识别特征**：
- 关键词：`@Transactional`、`synchronized`、`Lock`、`BEGIN TRANSACTION`、`COMMIT`
- 新增/删除分布式锁、数据库锁
- 事务传播级别变更（如 REQUIRED → REQUIRES_NEW）

**风险点**：
- 锁粒度变粗 → 性能下降或死锁
- 锁粒度变细 → 并发数据不一致
- 事务边界缩小 → 部分操作不在事务保护范围内

**测试建议**：
- 并发测试（同一用户多端操作）
- 验证异常回滚场景
- 压测验证锁等待超时配置

### 5. 安全相关变更

**识别特征**：
- 文件路径：`*Auth*`、`*Security*`、`*Permission*`、`*Token*`、`*Encrypt*`
- 关键词：`@PreAuthorize`、`hasRole`、权限校验、数据脱敏、加密解密

**风险点**：
- 权限校验条件放宽 → 越权访问
- 数据脱敏范围缩小 → 隐私数据泄露
- 加密算法变更 → 已有加密数据无法解密

**测试建议**：
- 无权限用户访问受保护接口（应返回 401/403）
- 验证数据脱敏字段在响应中已正确处理
- 测试不同角色的权限边界

## P1 中风险模式

### 6. 缓存策略变更

**识别特征**：
- 关键词：`@Cacheable`、`@CacheEvict`、`Redis`、`TTL`、`expire`
- 缓存 Key 格式变更、TTL 时间调整

**风险点**：
- 新旧 Key 格式共存 → 缓存穿透/双写问题
- TTL 缩短 → 数据库压力增加
- 缓存失效策略变更 → 脏数据残留

**测试建议**：
- 验证缓存命中和缓存未命中两种场景
- 验证缓存更新（写操作后缓存是否刷新）
- 发布时关注缓存预热策略

### 7. 外部服务依赖变更

**识别特征**：
- 新增 `FeignClient`、`RestTemplate`、`HttpClient` 调用
- 修改调用超时、重试配置
- 更换被调用的下游服务

**风险点**：
- 新依赖服务不稳定 → 级联超时
- 超时配置过短 → 误报错误
- 无熔断保护 → 雪崩风险

**测试建议**：
- Mock 下游服务异常场景（超时、500 错误）
- 验证熔断降级行为
- 压测新增调用链路的性能表现

### 8. 配置项变更

**识别特征**：
- `application.yml`、`application.properties`、`config.go`
- 新增配置项、修改默认值

**风险点**：
- 各环境配置不同步 → 环境差异导致 SIT 通过但 UAT 失败
- 配置缺失未做默认值保护 → 启动失败

**测试建议**：
- 确认 SIT/UAT/生产三套配置已同步
- 验证使用默认值时的行为是否符合预期

### 9. 批量/定时任务变更

**识别特征**：
- 关键词：`@Scheduled`、`XXL-JOB`、`cron`、批量处理、分片
- 修改执行频率、批次大小、处理逻辑

**风险点**：
- 执行时间窗口与业务高峰重叠 → 影响正常交易
- 批次过大 → 内存溢出或超时
- 处理逻辑错误 → 批量数据问题

**测试建议**：
- 手动触发任务执行，验证处理逻辑
- 用存量数据测试边界（空数据、大批量）
- 确认幂等性（任务重复执行不产生副作用）

## P2 低风险模式

### 10. 日志与监控变更

**识别特征**：
- 新增/修改 `log.info`、`log.error`、监控打点
- 日志格式调整、日志级别变更

**风险点**：
- 敏感信息被记录到日志 → 合规风险

**测试建议**：
- 确认日志不含密码、Token、身份证号等敏感信息

### 11. 代码重构（无业务变更）

**识别特征**：
- 仅命名变更、代码拆分/合并、提取方法
- 无功能逻辑改变

**风险点**：
- 重构引入意外行为变化（尤其是静态分析难以发现的边界）

**测试建议**：
- 执行已有自动化测试套件验证回归
- 重点关注被重构代码的核心业务场景

### 12. 依赖版本升级

**识别特征**：
- `pom.xml`、`go.mod`、`package.json` 中版本号变更

**风险点**：
- 依赖引入不兼容变更
- 传递依赖冲突

**测试建议**：
- 运行全量自动化测试
- 关注升级的依赖 Changelog 中的 Breaking Changes

## 优化需求专项分析

### 性能优化影响分析

**SQL 查询优化**：
- 查看新增/修改的索引（`EXPLAIN` 验证执行计划）
- 检查是否消除了全表扫描
- 验证分页参数处理（深分页问题）

**缓存引入**：
- 数据一致性策略（Cache-Aside vs Write-Through）
- 缓存预热方式
- 缓存击穿/雪崩防护

**异步化改造**：
- 接口响应由同步变异步 → 调用方需适配
- 消息队列消费失败的处理（重试、死信队列）

### 重构范围影响评估

**单服务内重构**（低影响）：
- 关注被重构方法的单元测试覆盖

**跨服务接口调整**（高影响）：
- 需通知所有下游服务测试兼容性
- 建议在不同时间窗口分批发布

**数据库层重构**（极高影响）：
- 字段/表重命名需要代码和 SQL 同步变更
- 建议使用"双写过渡"策略，不一次性删除旧字段
